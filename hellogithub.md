# Github Notes
- `git init` initializes a new project or your directory to be used
with Github (only used once)
- `git clone` create a working copy of a local repository by running the
command when using a remote server, your command will be
- `git clone username@host:/path/to/repository`
- `git add .` you can propose changes (add it to the Index) using
git add <filename> this is the first step in the basic git workflow.
- 'git push origin master` execute to send those changes to your remote repository, change master to whatever branch you want to push your changes to.
- `git remote add origin` <server>
now you are able to push your changes to the selected remote
server.
- `git status` the state of your working directory and staging area
which files arent being tracked by git and which ones are.
<br>
